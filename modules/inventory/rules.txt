WHAT THIS MODULE IS
-------------------
Complete inventory management: product catalog with SKUs, real-time
stock levels across multiple warehouses, stock adjustment tool, and
purchase orders tracked from draft through received.


ACTUAL FOLDER STRUCTURE
-----------------------
modules/inventory/

  package.json           ← own dependencies
  rules.txt              ← this file
  backend/
    router.ts            ← named export: { router }  (Express Router)
  frontend/
    layout.tsx           ← "use client" — tab nav for inventory/* pages
    page.tsx             ← /dashboard/inventory  (overview + KPI cards)
    products/
      page.tsx           ← /dashboard/inventory/products
    warehouses/
      page.tsx           ← /dashboard/inventory/warehouses

HOW THE ENGINE LOADS THIS MODULE
---------------------------------
1. At startup:  registerModules.ts reads manifest.json → registers manifest
2. On request:  /api/:moduleKey wildcard fires
3.              requireAuth → resolveTenant → attachHookService → requireModule("inventory") → router
4. Frontend routes appear in sidebar from manifest.json routes array

No migrations folder — tenant schema provisioned by orgProvisionService.


MANIFEST RULES (manifest.json)
--------------------------------
- key must be exactly: "inventory"
- requiredPlan: "growth"  (starter orgs cannot install this)
- Permission key format: inventory:{resource}:{action}
- roles values: "owner" | "manager" | "member"
- routes array must match every frontend page


BACKEND RULES (backend/router.ts)
-----------------------------------
Export pattern:
  import { Router } from "express";
  export const router = Router();

Available on req (set upstream — never re-apply in module):
  req.user        → authenticated user
  req.tenantDb    → org-scoped Prisma client
  req.orgCurrency → currency code string (e.g. "USD")
  req.hookService → fire webhooks + notifications

RBAC:
  GET  → open to all roles
  POST / PATCH / DELETE → guard with requireRole("manager") from "@business360/module-sdk"

Return shape:  { data } on success  |  { error: "message" } on failure
Status codes:  200 OK, 201 Created, 400 Bad Request, 403 Forbidden, 404 Not Found, 500

API endpoints:
  GET    /api/inventory/products              — list products (paginated, search by name/SKU)
  POST   /api/inventory/products              — create product  [manager+]
  GET    /api/inventory/products/:id          — get product with stock levels per warehouse
  PATCH  /api/inventory/products/:id          — update product  [manager+]
  DELETE /api/inventory/products/:id          — delete product  [manager+]

  GET    /api/inventory/warehouses            — list warehouses
  POST   /api/inventory/warehouses            — create warehouse  [manager+]
  PATCH  /api/inventory/warehouses/:id        — update warehouse  [manager+]
  DELETE /api/inventory/warehouses/:id        — delete warehouse  [manager+]

  POST   /api/inventory/stock/adjust          — adjust stock level (productId, warehouseId, delta)  [manager+]

  GET    /api/inventory/orders                — list purchase orders (paginated)
  POST   /api/inventory/orders                — create purchase order  [manager+]
  PATCH  /api/inventory/orders/:id            — update order / mark received  [manager+]
  DELETE /api/inventory/orders/:id            — delete order  [manager+]

  GET    /api/inventory/stats                 — KPIs: product count, low-stock count, order count

Zod validation: use err.message (NOT err.errors[0]) — root zod is v4 in this monorepo.


FRONTEND RULES (frontend/)
----------------------------
- layout.tsx: "use client" — uses usePathname() for active tab
- @/ resolves to apps/web/src/ — never import from apps/api/

Platform quirks (MUST follow):
- shadcn/ui uses @base-ui — NO asChild on <Button>
    RIGHT: <Link href="..." className={buttonVariants({ variant: "default" })}>...</Link>
- useSearchParams() requires <Suspense>
- Pages using useAuth() must be "use client"
- Tailwind only

Page content:
  /dashboard/inventory            — KPIs: total products, low-stock alerts, open orders, warehouse count
  /dashboard/inventory/products   — Table with SKU, price, total stock; stock adjust dialog; CRUD
  /dashboard/inventory/warehouses — Card grid of warehouses with stock summary; CRUD
  /dashboard/inventory/orders     — Table of purchase orders with status badge (draft/ordered/received); CRUD
