WHAT THIS MODULE IS
-------------------
Full accounting suite: invoices with line items, expense logging
by category, payment status tracking, and financial reporting.
Provides monthly revenue vs expenses data used by the Reports module.

-----------------------

HOW THE ENGINE LOADS THIS MODULE
---------------------------------
1. At startup:  registerModules.ts reads manifest.json → registers manifest
2. On request:  /api/:moduleKey wildcard fires
3.              requireAuth → resolveTenant → attachHookService → requireModule("accounting") → router
4. Frontend routes appear in sidebar from manifest.json routes array

No migrations folder — tenant schema provisioned by orgProvisionService.


MANIFEST RULES (manifest.json)
--------------------------------
- key must be exactly: "accounting"
- requiredPlan: "pro"
- Permission key format: accounting:{resource}:{action}
- roles values: "owner" | "manager" | "member"
- routes array must match every frontend page


BACKEND RULES (backend/router.ts)
-----------------------------------
Export pattern:
  import { Router } from "express";
  export const router = Router();

Available on req (set upstream — never re-apply in module):
  req.user        → authenticated user
  req.tenantDb    → org-scoped Prisma client
  req.orgCurrency → currency code string
  req.hookService → fire webhooks + notifications

RBAC:
  GET  → open to all roles
  POST / PATCH / DELETE → guard with requireRole("manager") from "@business360/module-sdk"

Return shape:  { data } on success  |  { error: "message" } on failure
Status codes:  200 OK, 201 Created, 400 Bad Request, 403 Forbidden, 404 Not Found, 500

API endpoints:
  GET    /api/accounting/invoices             — list invoices (paginated, filter by status)
  POST   /api/accounting/invoices             — create invoice with line items  [manager+]
  GET    /api/accounting/invoices/:id         — get invoice + line items
  PATCH  /api/accounting/invoices/:id         — update invoice / change status  [manager+]
  DELETE /api/accounting/invoices/:id         — delete invoice  [manager+]

  GET    /api/accounting/expenses             — list expenses (paginated, filter by category)
  POST   /api/accounting/expenses             — create expense  [manager+]
  PATCH  /api/accounting/expenses/:id         — update expense  [manager+]
  DELETE /api/accounting/expenses/:id         — delete expense  [manager+]

  GET    /api/accounting/stats                — KPIs: total revenue, total expenses, outstanding, overdue count
  GET    /api/accounting/monthly              — monthly revenue vs expenses (last 12 months) — used by Reports

Invoice status (string enum): draft → sent → paid | overdue | cancelled
Expense category examples: travel, software, office, marketing, other

Zod validation: use err.message (NOT err.errors[0]) — root zod is v4 in this monorepo.


FRONTEND RULES (frontend/)
----------------------------
- layout.tsx: "use client" — uses usePathname() for active tab
- @/ resolves to apps/web/src/ — never import from apps/api/

Platform quirks (MUST follow):
- shadcn/ui uses @base-ui — NO asChild on <Button>
    RIGHT: <Link href="..." className={buttonVariants({ variant: "default" })}>...</Link>
- useSearchParams() requires <Suspense>
- Pages using useAuth() must be "use client"
- Tailwind only

Page content:
  /dashboard/accounting          — KPIs: total revenue, expenses, outstanding invoices, overdue count
  /dashboard/accounting/invoices — Table with status badge; create invoice dialog (line items); mark paid
  /dashboard/accounting/expenses — Table with category badge; add expense dialog; edit/delete
