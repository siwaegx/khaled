MANIFEST RULES
--------------
- key must be exactly: "projects"
- version: start "1.0.0"
- routes array must match every page in frontend/pages/
- permissions must cover all exposed operations
- projects:tasks:write allows members (team collaboration feature)
- projects:write — owner and admin only (create/delete projects)


BACKEND RULES
-------------
- All routes protected by requireAuth — never skip
- Use tenant DB — never write project data to shared DB
- Logic in services/, not route handlers
- Return consistent JSON:  { data, error, message }
- HTTP status codes: 200, 201, 400, 401, 403, 404, 500
- TypeScript strict — no "any"
- Max 300 lines per file

Entities and their API endpoints:

  Projects
    GET    /api/projects                  → list projects (filter by status)
    POST   /api/projects                  → create project
    GET    /api/projects/:id              → get project with task summary and milestones
    PATCH  /api/projects/:id              → update project (name, description, status)
    DELETE /api/projects/:id              → archive project (set status = archived)
    Fields: id, name, description, status, ownerId, startDate, dueDate?, createdAt
    Status enum: active | on_hold | completed | archived

  Tasks
    GET    /api/projects/tasks            → list tasks (filter by project, assignee, status)
    POST   /api/projects/tasks            → create task
    GET    /api/projects/tasks/:id        → get task with comments
    PATCH  /api/projects/tasks/:id        → update task (title, status, assignee, priority)
    DELETE /api/projects/tasks/:id        → delete task
    Fields: id, projectId, title, description, status, priority, assigneeId?,
            dueDate?, completedAt?, createdAt
    Status enum: todo | in_progress | review | done
    Priority enum: low | medium | high | urgent

  Milestones
    GET    /api/projects/:id/milestones   → list milestones for a project
    POST   /api/projects/:id/milestones   → create milestone
    PATCH  /api/projects/milestones/:id   → update milestone (title, dueDate, completed)
    DELETE /api/projects/milestones/:id   → delete milestone
    Fields: id, projectId, title, dueDate, completed, completedAt?

  Business rules:
    - Task completion sets completedAt timestamp automatically (server-side)
    - Project completion = all milestones completed (can also be manually set)
    - Tasks without a project are not allowed (projectId required)
    - Deleting a project does NOT delete tasks — set tasks.projectId = null or archive them

DATABASE MIGRATION (migrations/001_projects_initial.sql)
--------------------------------------------------------
Runs inside TENANT database only.

Tables:
  proj_projects    (id, name, description TEXT, status, owner_id, start_date DATE,
                    due_date DATE, created_at, updated_at)
                   status CHECK IN ('active','on_hold','completed','archived')
  proj_tasks       (id, project_id REFERENCES proj_projects, title, description TEXT,
                    status, priority, assignee_id, due_date DATE, completed_at TIMESTAMPTZ,
                    created_at, updated_at)
                   status CHECK IN ('todo','in_progress','review','done')
                   priority CHECK IN ('low','medium','high','urgent')
  proj_milestones  (id, project_id REFERENCES proj_projects, title, due_date DATE,
                    completed BOOLEAN DEFAULT false, completed_at TIMESTAMPTZ,
                    created_at, updated_at)

Rules:
- id: TEXT PRIMARY KEY DEFAULT gen_random_uuid()
- Timestamps: TIMESTAMPTZ DEFAULT NOW()
- proj_tasks and proj_milestones: CASCADE on proj_projects delete is optional
  (prefer SET NULL to preserve task history after project archive)
- No org_id column
- Migration idempotent: CREATE TABLE IF NOT EXISTS


INSTALLATION FLOW

2. Engine checks InstalledModule — rejects if already active
3. Engine runs 001_projects_initial.sql on tenant DB
4. InstalledModule saved { orgId, moduleKey: "projects", isActive: true }
5. Routes registered under /api/projects/*
6. Sidebar updated from manifest.routes

Uninstall:
- isActive = false (data preserved)
- Routes deregistered
