WHAT THIS MODULE IS
-------------------
Project management for agile teams. Projects have statuses and
contain tasks with priorities, assignees, and due dates. Provides
a cross-project task view and project lifecycle tracking.


ACTUAL FOLDER STRUCTURE
-----------------------
modules/projects/
  manifest.json          ← JSON metadata
  package.json           ← own dependencies
  rules.txt              ← this file
  backend/
    router.ts            ← named export: { router }  (Express Router)
  frontend/
    layout.tsx           ← "use client" — tab nav for projects/* pages
    page.tsx             ← /dashboard/projects  (projects list + overview)
    tasks/
      page.tsx           ← /dashboard/projects/tasks

Thin re-exports (do not edit):
  apps/web/src/app/dashboard/projects/page.tsx
  apps/web/src/app/dashboard/projects/layout.tsx
  apps/web/src/app/dashboard/projects/tasks/page.tsx


HOW THE ENGINE LOADS THIS MODULE
---------------------------------
1. At startup:  registerModules.ts reads manifest.json → registers manifest
2. On request:  /api/:moduleKey wildcard fires
3.              requireAuth → resolveTenant → attachHookService → requireModule("projects") → router
4. Frontend routes appear in sidebar from manifest.json routes array

No migrations folder — tenant schema provisioned by orgProvisionService.


MANIFEST RULES (manifest.json)
--------------------------------
- key must be exactly: "projects"
- requiredPlan: "enterprise"
- Permission key format: projects:{resource}:{action}
- roles values: "owner" | "manager" | "member"
- routes array must match every frontend page


BACKEND RULES (backend/router.ts)
-----------------------------------
Export pattern:
  import { Router } from "express";
  export const router = Router();

Available on req (set upstream — never re-apply in module):
  req.user        → authenticated user
  req.tenantDb    → org-scoped Prisma client
  req.hookService → fire webhooks + notifications

RBAC — Tasks have a special rule (members can edit their own tasks):
  GET  → open to all roles
  POST / DELETE projects → manager+ only
  PATCH projects/:id → manager+ only
  POST / PATCH / DELETE tasks → open to members (projects:tasks:write permission)
    Exception: deleting a project (and cascading its tasks) is manager+ only

Return shape:  { data } on success  |  { error: "message" } on failure
Status codes:  200 OK, 201 Created, 400 Bad Request, 403 Forbidden, 404 Not Found, 500

API endpoints:
  GET    /api/projects                    — list projects (paginated)
  POST   /api/projects                    — create project  [manager+]
  GET    /api/projects/:id                — get project + task summary
  PATCH  /api/projects/:id                — update project  [manager+]
  DELETE /api/projects/:id                — delete project + tasks  [manager+]

  GET    /api/projects/tasks              — list all tasks across projects (paginated, filter by assignee/project)
  POST   /api/projects/tasks              — create task  [member+]
  PATCH  /api/projects/tasks/:id          — update task (status, priority, assignee, due date)  [member+]
  DELETE /api/projects/tasks/:id          — delete task  [manager+]

  GET    /api/projects/stats              — KPIs: active projects, total tasks, overdue tasks, completed tasks

Project status (string enum): planning → active → on_hold → completed | cancelled
Task priority (string enum): low | medium | high | urgent
Task status (string enum): todo → in_progress → review → done

Zod validation: use err.message (NOT err.errors[0]) — root zod is v4 in this monorepo.


FRONTEND RULES (frontend/)
----------------------------
- layout.tsx: "use client" — uses usePathname() for active tab
- @/ resolves to apps/web/src/ — never import from apps/api/

Platform quirks (MUST follow):
- shadcn/ui uses @base-ui — NO asChild on <Button>
    RIGHT: <Link href="..." className={buttonVariants({ variant: "default" })}>...</Link>
- useSearchParams() requires <Suspense>
- Pages using useAuth() must be "use client"
- Tailwind only

Page content:
  /dashboard/projects       — Project cards with status badge, task count; create project dialog; CRUD
  /dashboard/projects/tasks — Table of all tasks: project name, assignee, priority badge, due date; filter by project
