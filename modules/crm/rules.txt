WHAT THIS MODULE IS
-------------------
Tracks the full sales pipeline: leads → deals → customers.
Supports lead conversion to customers, kanban deal stages,
and fires webhook hooks on lead.created / lead.updated events.

HOW THE ENGINE LOADS THIS MODULE
---------------------------------
1. At startup:  registerModules.ts reads manifest.json → registers manifest
2. On request:  /api/:moduleKey wildcard in route_registry.ts fires
3.              requireAuth   → checks JWT
4.              resolveTenant → attaches req.tenantDb (org-scoped Prisma client)
5.              attachHookService → attaches req.hookService for webhook/notification firing
6.              requireModule("crm") → confirms org has crm installed + active
7.              loadRouter("crm") → require()s backend/router.ts (cached after first load)
8. Frontend routes appear in sidebar automatically from manifest.json routes array

No migrations folder — tenant schema is provisioned by orgProvisionService at org creation.
No InstalledModule table — install state tracked in Organization.installedModules (JSON).


MANIFEST RULES (manifest.json)
--------------------------------
- key must be exactly: "crm"
- requiredPlan: "starter"  (engine rejects install if org.plan < starter)
- routes array: every path must match a frontend page file
- permissions array: every key must be enforced in backend middleware
- Permission key format: crm:{resource}:{action}
- roles values: "owner" | "manager" | "member"  (never use "admin")
- version: semver — bump minor for features, patch for fixes


BACKEND RULES (backend/router.ts)
-----------------------------------
Export pattern:
  import { Router } from "express";
  export const router = Router();

Authentication & tenant:
  - req.user    → authenticated user (set by requireAuth upstream)
  - req.tenantDb → org-scoped Prisma client (set by resolveTenant upstream)
  - req.hookService.fireHook(event, payload) → fires webhooks + notifications
  - Never import requireAuth or resolveTenant inside a module router (already applied)

RBAC (applied per route group, not globally):
  GET  routes → open to all roles (owner, manager, member)
  POST / PATCH / DELETE → import { requireRole } from "@business360/module-sdk"
                          and guard with router.use(requireRole("manager"))
                          or per-route for fine-grained control

Return shape:  { data } on success  |  { error: "message" } on failure
Status codes:  200 OK, 201 Created, 400 Bad Request, 403 Forbidden, 404 Not Found, 500 Server Error

API endpoints:
  GET    /api/crm/leads           — list leads (paginated, search by name/email)
  POST   /api/crm/leads           — create lead  [manager+]
  GET    /api/crm/leads/:id       — get lead
  PATCH  /api/crm/leads/:id       — update lead  [manager+]  → fires lead.updated hook
  DELETE /api/crm/leads/:id       — delete lead  [manager+]
  POST   /api/crm/leads/:id/convert — convert lead to customer [manager+] → fires lead.created hook

  GET    /api/crm/deals           — list deals
  POST   /api/crm/deals           — create deal  [manager+]
  PATCH  /api/crm/deals/:id       — update deal (incl. stage change)  [manager+]
  DELETE /api/crm/deals/:id       — delete deal  [manager+]

  GET    /api/crm/customers       — list customers
  POST   /api/crm/customers       — create customer  [manager+]
  GET    /api/crm/customers/:id   — get customer + linked deals
  PATCH  /api/crm/customers/:id   — update customer  [manager+]
  DELETE /api/crm/customers/:id   — delete customer  [manager+]

  GET    /api/crm/stats           — summary KPIs (lead count, deal value, customer count)

Deal stages (string enum): lead → qualified → proposal → negotiation → won | lost
Zod validation: use err.message (NOT err.errors[0]) — root zod is v4 in this monorepo.


FRONTEND RULES (frontend/)
----------------------------
- layout.tsx: "use client" directive required — uses usePathname() for active tab
- page.tsx and sub-pages: can be server or client components
- Import alias @/ resolves to apps/web/src/ (configured in apps/web/tsconfig.json)
- Never import from apps/api/ — frontend is completely separate from backend

Platform quirks (MUST follow):
- shadcn/ui uses @base-ui — NO asChild prop on <Button>
    WRONG: <Button asChild><Link href="...">Go</Link></Button>
    RIGHT: <Link href="..." className={buttonVariants({ variant: "default" })}>Go</Link>
- useSearchParams() requires <Suspense> — extract the reading component as a child
- Pages using useAuth() must be "use client"
- Tailwind only — no inline styles, no hardcoded color values
- Max 300 lines per file — split into components if exceeded

Page content:
  /dashboard/crm        — KPI cards: total leads, open deals, customers, deal value; recent activity
  /dashboard/crm/leads  — Table: status badge, pagination, search; "New Lead" dialog
  /dashboard/crm/deals  — Kanban board matching Stages enum; drag to change stage; card shows title + value
  /dashboard/crm/customers — Table with company column, linked deal count; click → detail with deal history


HOOKS FIRED BY THIS MODULE
----------------------------
  req.hookService.fireHook("lead.created", { leadId, name, orgId })
  req.hookService.fireHook("lead.updated", { leadId, name, status, orgId })
These fire webhooks registered in OrgWebhook and create in-app Notification records.
