WHAT THIS MODULE IS
-------------------
Unified analytics across all installed modules. Aggregates KPIs
from CRM, Inventory, Accounting, HR, and Projects into one view.
Provides monthly revenue vs expenses chart and alert summaries
(overdue invoices, low stock). Read-only — no data mutations here.


ACTUAL FOLDER STRUCTURE
-----------------------
modules/reports/
  manifest.json          ← JSON metadata
  package.json           ← own dependencies
  rules.txt              ← this file
  backend/
    router.ts            ← named export: { router }  (Express Router)
  frontend/
    page.tsx             ← /dashboard/reports  (single page, no sub-tabs)

Note: No layout.tsx — reports has only one page, no tab navigation.

Thin re-export (do not edit):
  apps/web/src/app/dashboard/reports/page.tsx


HOW THE ENGINE LOADS THIS MODULE
---------------------------------
1. At startup:  registerModules.ts reads manifest.json → registers manifest
2. On request:  /api/:moduleKey wildcard fires
3.              requireAuth → resolveTenant → attachHookService → requireModule("reports") → router
4. Frontend routes appear in sidebar from manifest.json routes array

No migrations folder — reports reads from existing tenant tables (no own schema).


MANIFEST RULES (manifest.json)
--------------------------------
- key must be exactly: "reports"
- requiredPlan: "enterprise"
- Permission key format: reports:{action}
- roles values: "owner" | "manager" | "member"
- routes array: single entry for /dashboard/reports


BACKEND RULES (backend/router.ts)
-----------------------------------
Export pattern:
  import { Router } from "express";
  export const router = Router();

IMPORTANT: This router has NO auth middleware baked in (it was intentionally left out
at the route level because auth is already enforced upstream by the wildcard handler).
Do not add requireAuth inside this router.

Available on req (set upstream — never re-apply):
  req.user        → authenticated user
  req.tenantDb    → org-scoped Prisma client

RBAC: All reports endpoints are read-only — no mutations, no requireRole needed.
All roles (owner, manager, member) can view reports.

Return shape:  { data } on success  |  { error: "message" } on failure
Status codes:  200 OK, 400 Bad Request, 403 Forbidden, 500

API endpoints:
  GET    /api/reports/summary              — KPI cards from all modules:
                                              CRM: lead count, customer count
                                              Inventory: product count, low-stock count
                                              Accounting: total revenue, total expenses, overdue invoices
                                              HR: employee count, pending leave
                                              Projects: active projects, overdue tasks

  GET    /api/reports/revenue              — Monthly revenue vs expenses (last 12 months)
                                              Pulls from accounting tables only
                                              Returns: [{ month, revenue, expenses }]

  GET    /api/reports/alerts               — Alert list:
                                              overdue invoices (accounting.invoices where status=overdue)
                                              low-stock products (inventory stock < threshold)
                                              Returns: [{ type, message, severity }]

When a module is not installed, its stats should return 0 / empty — use try/catch per module query.

Zod validation: use err.message (NOT err.errors[0]) — root zod is v4 in this monorepo.


FRONTEND RULES (frontend/)
----------------------------
- page.tsx: can be server or client component
- No layout.tsx needed (single page module)
- @/ resolves to apps/web/src/ — never import from apps/api/

Platform quirks:
- Tailwind only — no inline styles
- Render KPI cards in a responsive grid (2 cols mobile, 4 cols desktop)
- Monthly chart: table or simple bar chart; avoid heavy charting libraries

Page content:
  /dashboard/reports — KPI summary cards (grouped by module), monthly P&L table, alerts list
