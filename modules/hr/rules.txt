MANIFEST RULES
--------------
- key must be exactly: "hr"
- version: start "1.0.0"
- routes array must match every page in frontend/pages/
- permissions must cover all exposed operations
- hr:contracts:read and hr:contracts:write — owner and admin only (sensitive data)
- hr:employees:read — all roles including member


BACKEND RULES
-------------
- All routes protected by requireAuth — never skip
- Use tenant DB — employee/contract data never goes to shared DB
- Logic in services/, not route handlers
- Return consistent JSON:  { data, error, message }
- HTTP status codes: 200, 201, 400, 401, 403, 404, 500
- TypeScript strict — no "any"
- Max 300 lines per file

Entities and their API endpoints:

  Employees
    GET    /api/hr/employees           → list employees (filter by department, status)
    POST   /api/hr/employees           → add employee
    GET    /api/hr/employees/:id       → get employee with leave summary
    PATCH  /api/hr/employees/:id       → update employee info
    DELETE /api/hr/employees/:id       → soft-delete (set active = false)
    Fields: id, firstName, lastName, email, phone, department, position,
            startDate, salary?, active, createdAt

  Leave Requests
    GET    /api/hr/leave               → list leave requests (filter by employee, status)
    POST   /api/hr/leave               → submit leave request
    GET    /api/hr/leave/:id           → get request details
    PATCH  /api/hr/leave/:id/approve   → approve (admin/owner only)
    PATCH  /api/hr/leave/:id/reject    → reject with reason (admin/owner only)
    Fields: id, employeeId, type, startDate, endDate, reason, status, reviewedBy?, reviewNote?
    Leave types enum: annual | sick | unpaid | other
    Status enum: pending → approved | rejected

  Contracts
    GET    /api/hr/contracts           → list contracts (admin/owner only)
    POST   /api/hr/contracts           → create contract
    GET    /api/hr/contracts/:id       → get contract
    PATCH  /api/hr/contracts/:id       → update contract
    Fields: id, employeeId, type, startDate, endDate?, salary, currency, signedAt?, active
    Contract type enum: full_time | part_time | freelance | internship

  Business rules:
    - Salary field is sensitive — only owner/admin can see it (enforce in service layer)
    - Leave overlap check: reject if employee already has approved leave on same dates
    - Soft-delete employees (set active=false) — do not DELETE rows (preserve history)


FRONTEND RULES
--------------
Follow style.txt for all UI. HR-specific rules:

- Dashboard page (/dashboard/hr):
    KPI cards: total employees, pending leave requests, active contracts, new this month
    Department breakdown bar chart

- Employees page (/dashboard/hr/employees):
    Table or card grid with avatar initials, name, department badge, position
    Filter by department, search by name/email
    "Add Employee" button → form modal
    Status indicator: active (green dot) / inactive (gray)

- Leave page (/dashboard/hr/leave):
    Table: employee name, type badge, date range, status badge
    Admin view: approve/reject buttons on pending rows
    Member view: own requests only (filter by session user's employee record)
    "Request Leave" button → form with type, dates, reason

- Contracts page (/dashboard/hr/contracts):
    Table: employee, type, start date, end date, salary (admin/owner only)
    Visible to admin/owner only — hide from members (check permission in frontend)

- "use client" on any file using hooks

DATABASE MIGRATION (migrations/001_hr_initial.sql)
--------------------------------------------------
Runs inside TENANT database only.

Tables:
  hr_employees   (id, first_name, last_name, email UNIQUE, phone, department, position,
                  start_date, salary NUMERIC(12,2), active BOOLEAN DEFAULT true,
                  created_at, updated_at)
  hr_leave       (id, employee_id REFERENCES hr_employees, type, start_date, end_date,
                  reason TEXT, status, reviewed_by, review_note, created_at, updated_at)
                  status CHECK IN ('pending','approved','rejected')
                  type CHECK IN ('annual','sick','unpaid','other')
  hr_contracts   (id, employee_id REFERENCES hr_employees, type, start_date, end_date,
                  salary NUMERIC(12,2), currency CHAR(3) DEFAULT 'USD',
                  signed_at TIMESTAMPTZ, active BOOLEAN DEFAULT true, created_at, updated_at)
                  type CHECK IN ('full_time','part_time','freelance','internship')

Rules:
- id: TEXT PRIMARY KEY DEFAULT gen_random_uuid()
- Timestamps: TIMESTAMPTZ DEFAULT NOW()
- hr_leave and hr_contracts: CASCADE on hr_employees delete
- No org_id column
- Migration idempotent: CREATE TABLE IF NOT EXISTS

Uninstall:
- isActive = false (data preserved)
- Routes deregistered
